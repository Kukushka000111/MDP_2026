# Сценарий демонстрации на защите (5–7 минут)

Подготовка: PostgreSQL запущен, `server/.env` настроен, в корне выполнены `npm install`, `npm run db:migrate`, `npm --workspace server run db:seed`. Два терминала: `npm run dev:server` и `npm run dev:client`. Сайт: **http://localhost:5173**.

## 1. Гость и лента (≈1 мин)

1. Открыть ленту без входа.
2. Показать фильтры: категория, район, **«Официальные / От жителей»**, даты.
3. Клик по карточке или маркеру на карте → страница события.
4. Добавить в избранное (localStorage) — после входа синхронизируется.

## 2. Регистрация и вход (≈1,5 мин)

1. В шапке **«Регистрация»** → отдельная страница `#register`.
2. Поля: имя, фамилия, email, логин, пароль, подтверждение, чекбокс правил, возраст 18+, пол.
3. Показать проверку email/логина в БД (подсказки «свободен» / «занят»).
4. После регистрации — **«Аккаунт»**: телефон, Telegram, VK, фото.
5. **«Вход»** `#login` — по логину и паролю (JWT на 1 день).

## 3. Создание события от жителя (≈1,5 мин)

1. Форма «Создать мероприятие»: организатор и контакты **подставляются из профиля** (не вводятся вручную).
2. Адрес `Омск, …, дом`, лимит мест (по желанию).
2. Геокодинг или клик по карте.
3. Отправить → toast «на модерации».
4. Раздел **«Мои мероприятия»** → статус «На модерации».

## 4. Модерация (админ) (≈1,5 мин)

1. Выйти, войти под email из `ADMIN_EMAILS` (или заранее зарегистрированный админ).
2. **«Модерация»** → очередь, комментарий, «Одобрить» / «Нужны правки» / «Отклонить».
3. Войти обратно как автор → блок **«Уведомления о модерации»** с текстом решения и комментарием.

## 5. Публикация и участие (≈1 мин)

1. Одобренное событие в ленте (фильтр «От жителей»).
2. «Записаться» на мероприятие.
3. Страница события → **кликабельные контакты** организатора.

## 6. Отзыв (по желанию)

Если в БД событие с `ends_at` в прошлом — оставить отзыв на странице события.

## Резервные формулировки

- **Премодерация** — community-события не попадают в ленту до `APPROVED`.
- **Геокодинг** — через бэкенд (Nominatim), обход CORS.
- **Стек** — React + Vite, Express + PostgreSQL, Leaflet.
